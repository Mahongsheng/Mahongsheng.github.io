如何运行跨境电商平台？

# 1. 克隆代码库

首先，您需要将后端代码库克隆到你的电脑中：

git clone https://github.com/Mahongsheng/e-commerce.git
1

克隆完毕后，推荐使用IntelliJ IDEA打开本项目，打开后IDEA将会默认使用内置的Maven下载依赖并索引整个项目，耐心等待即可，如果太慢，可以修改Maven镜像源为国内地址。

# 2. 搭建Eureka Server

Eureka Server提供服务注册服务，各个节点启动后，会在Eureka Server中进行注册，这样Eureka Server中的服务注册表中将会存储所有可用服务节点的信息，服务节点的信息可以在界面中直观的看到。

Eureka Server本身也是一个服务，默认情况下会自动注册到Eureka注册中心。

在此我们搭建单机版的Eureka Server，您可以使用我打包好的Jar包或者自行创建并打包Eureka Server（网上一大堆啦，Google下~）

Eureka Server Jar包百度网盘下载链接：https://pan.baidu.com/s/1PO7XO_DjewO4nhGzGWGCSA 提取码：fm1t

下载完毕后，您可以选择在自己的云服务器上部署，也可以选择在本机部署：

  • 不使用Docker：直接java -jar eureka-server.jar，运行后会暴露8761端口；

  • 使用Docker：创建Dockerfile并将eureka-server.jar放到和Dockerfile同级别目录下：

    FROM java:8
VOLUME /eureka-server
ADD eureka-server-1.0.jar eureka-server.jar
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/./urandom","-jar","/eureka-server.jar"]
    1
    2
    3
    4

    创建完毕后，运行：

    docker build -t eureka-server-image .
    1

    此时Docker会自动创建eureka-server-image镜像，再运行：

    docker run -d -p 8761:8761 eureka-server-image
    1

    此时会运行该镜像，生成容器并暴露8761端口。

注意：在此我们需要关注Eureka的IP地址和端口，之后在配置文件中会用到。

# 3. 搭建MySQL数据库

在本项目中，我们直接在阿里云上购买了云数据库MySQL，由于账号与密码不方便透露，因此在此您需要自行搭建MySQL数据库。

在项目目录others文件夹中，我们已经提供了SQL文件，您只需要导入到自己的MySQL数据库即可，注意MySQL数据库的版本应为8以上。

# 4. 搭建Redis

在本项目，同上我们也在阿里云上购买了云缓存Redis，在此可以将地址与密码告知大家，但其将于2022年中下旬过期，因此我们还是推荐您使用自己的Redis，在后期配置中心的讲解中，我们会直接使用云缓存Redis。

搭建Redis是比较简单的，本项目中主要在登录界面的验证码处使用Redis存储以达到验证码过期的效果。

# 5. 创建配置文件仓库

在e-commerce项目中，我们使用到了统一的配置中心，您可以通过这个博客 (opens new window)来进行了解，配置文件仓库的目录结构如图所示：

(opens new window)

您需要在github或gitee（推荐）等平台创建仓库，并存放统一的配置文件，这样就达到了将配置与项目代码分隔的作用。

搭建完毕配置文件仓库后，您必须修改e-commerce-config-server模块中的application.yaml文件：

server:
  port: 8888

eureka:
  client:
    serviceUrl:
      register-with-eureka: true
      fetch-registry: true
      defaultZone: http://106.13.127.90:8761/eureka # 修改为你的Eureka Server地址
  instance:
    preferIpAddress: true
    instance-id: ${spring.cloud.client.ip-address}:${server.port}

spring:
  application:
    name: e-commerce-config-server  # 应用名称
  cloud:
    config:
      server:
        git:
          uri: https://gitee.com/xxx #配置文件所在仓库
          default-label: master # 配置文件分支
          search-paths: config # 配置文件所在根目录
          username: xxx # 用户名
          password: xxx # 密码

# 6. 运行！

请注意运行顺序：

  1. 请先启动配置中心e-commerce-config-server；
  2. 在所有微服务模块中，请先启动e-commerce-admin，这个模块为登录、权限验证模块，少了它验证码刷不出来登录不进去系统；
  3. 接下来随意运行其他微服务模块吧~
  4. 运行前端项目观察效果吧~
    1. npm install --registry=https://registry.npm.taobao.org这样install可以保证使用国内镜像，肥肠快啦
    2. npm run dev跑起来！
  5. 用户名和密码：
    • admin string
    • bvo string
    • mvo string

# 7. 项目打包

在观察项目结构的过程中，您可能会发现，模块之间是有引用的，比如e-commerce-mybatis模块作为数据库连接层，被所有其他微服务模块所引用，因此打包时需要注意先后顺序：

  1. 请先打包e-commerce-common模块，运行mvn clean与mvn install，install是为了将该jar包存入本地仓库，否则打包其他模块时如果用到了该模块，会显示找不到~
  2. 先后打包e-commerce-mybatis、e-commerce-redis、e-commerce-security，方式同1，必须用mvn install；
  3. 接着直接打包各个微服务模块就好了~运行mvn clean与mvn package，这里不需要install，原因同1；

打包好后，我们就可以看到target下的jar包了~冲冲冲~

# 8. 使用Docker部署

细心的您一定发现每个微服务模块下都有一个Dockerfile，没错我们期望使用Docker部署这几个微服务模块（不使用Docker的话，如果您只有一台虚拟机或者低性能云服务，担心跑不起来或速度太慢）

在创建Docker镜像之前一定确保对项目打包完毕，否则莫得jar包。

同搭建Eureka Server时使用Docker的步骤，在Dockerfile同级目录打开终端，运行：

docker build -t e-commerce-admin .

docker run -d -p 9010:9010 e-commerce-admin
