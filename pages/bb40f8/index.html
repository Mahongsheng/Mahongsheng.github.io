百度-质量效能研发部-实习总结

# 百度-质量效能研发部-实习总结

作为全栈开发工程师，开发大前端自动化测试工具，包括Web Server端、执行端、前端，并进行日常运维；

# 一、全栈研发

  1. 开发前端

    • 使用Vue.js框架与Ant Design Vue组件库；

    • 使用ECharts绘制图表；

    • 将项目从旧前端迁移到新前端；

  2. 开发Web Server端

    • 使用Dubbo、SpringBoot框架完成常规接口开发；

    • 使用Quartz框架，开发动态定时任务；

    • 编写复杂SQL生成价值模型等统计数据；

    • 参与分库分表设计与实现，合理构建数据库索引；

  3. 开发执行器

    • 使用Appium移动自动化测试框架开发测试用例动作；

    • 使用WebSocket与ConcurrentHashMap向前端实时推送执行数据；

    • 参与智能测试方案设计与研发，阅读相关论文，为修复失效的用例，使用DFS实现安卓控件有向图的生成以及两点间可达路径的查找；

  4. 独立负责统计端

    • 使用SpringBoot框架、undertow服务器；

    • 提供系统运行数据查询接口；

    • 定时向工作群推送稳定数据和报警信息，并引入Spring-Retry重试机制保证信息送达；

    • 使用Screw生成数据库文档；

  5. 工具

    • IDEA, VS Code, Hbuilder, Iterm, zsh, ApiPost, Dbear

# 二、运维

  1. Web Server与统计端测试环境上线：SpringBoot工程打包、编写部署脚本、登录堡垒机、登录测试机、执行上线操作；
  2. 通过流水线进行前端上线；

# 三、系统数据

  1. 总用例数23000左右；
  2. 用户400左右；
  3. PC端日均10万，移动端日均4万；

# 四、个人数据

  1. 总提交357次；
  2. 参与两个项目，FAST与NoCase；

# 五、总结

负责大前端智能测试云平台的服务端、执行端与统计端开发与维护。

  • 负责服务端业务接口的代码设计与开发。为客制化测试任务，在集群中使用Quartz与Corn开发动态定时任务；为生成平台价值模型，在MySQL从库中编写复杂SQL以获得统计数据；
  • 负责执行端测试动作集的代码设计与开发。使用Selenium开发常规页面动作；为自动生成GUI控件拓扑图以便自动修复测试用例，复现相关论文，使用DFS生成安卓控件有向图并搜索控件间可达路径；
  • 独立负责统计端的设计与开发。基于Springboot与Undertow技术开发系统运行数据查询接口；使用@Schedule注解向如流机器人定时推送数据和报警信息，引入Spring-Retry重试机制以保证可靠交付；

Quartz如何保证分布式执行任务不重复？

Quartz官方分布式方案，需要根据官方文档新建数据表；

  1. 在数据库中创建quartz需要的表。这些表由quartz自己维护，用于保存数据和维护程序运行状态，这些表对于程序员而言是透明的，不需要关注，只需创建好表即可。
  2. 在工程的src下增加quartz.properties配置文件，进行集群相关操作的配置。
  3. QRTZ_LOCKS就是Quartz集群实现同步机制的行锁表，例如htc_scheduler集群下的锁：
    1. 可以看出QRTZ_LOCKS中有5条记录，代表5把锁，分别用于实现多个Quartz Node对Job、Trigger、Calendar访问的同步控制。
